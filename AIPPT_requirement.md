> 本文档描述 PPT 编辑器中"AI 助手（子言）"功能模块的产品需求。
> 编辑器本身（元素编辑、放映、导入导出等）为系统已有能力，不在本文档范围内。
> AI 助手通过调用编辑器已有的 Store API 来操作 PPT 元素。

---

## 文档目的

AI 助手"子言"是嵌入在线 PPT 编辑器右侧工具栏中的智能对话模块。教师通过自然语言与 AI 对话，AI 可以直接操作 PPT 元素（修改格式、添加内容、新建页面），也可以生成教学内容供用户选择性插入。本文档聚焦 AI 助手的交互逻辑、对话管理、内容生成与插入规则。

---

## 第 1 章：产品概述

### 1.1 功能定位

AI 助手是 PPT 编辑器的增值能力层，不改变编辑器的核心交互。它以"侧边栏 tab + 悬浮对话框"的形态存在，用户可以随时在编辑属性和 AI 对话之间切换，互不干扰。

### 1.2 核心场景

| 场景 | 用户行为 | AI 响应 |
|------|----------|---------|
| 生成教学内容 | "帮我设计一段课堂引入" | 生成 Markdown 格式内容，用户可插入 PPT |
| 调整元素格式 | 选中文字框 → "字号改大，颜色改红" | 直接修改元素属性，回复"已调整"+撤销按钮 |
| 批量操作 | "把背景改成深蓝色" | 调用 update_slide_background 工具 |
| 生成例题 | 选择"例题生成"工具 → 输入需求 | 生成题目内容，可插入当前页或新建页 |
| 生成演讲稿 | 选择"生成演讲稿"工具 | 生成演讲稿，可插入到页面备注 |

### 1.3 目标用户

教师和教研人员，需要高效制作课件，不熟悉 PPT 排版细节，希望通过自然语言完成内容生成和格式调整。

## 第 2 章：入口与布局

### 2.1 AI 助手入口

AI 助手作为右侧工具栏的一个 tab，与"设计/样式/位置/动画"等 tab 并列。

| 位置 | 说明 |
|------|------|
| 右侧工具栏竖向 tab 栏 | 紧挨"动画"tab 下方，有分隔线 |
| 图标 | 星形（sparkle）线框图标，科技感 |
| 标签文字 | "AI"，渐变色（紫→蓝） |

### 2.2 Tab 切换规则

AI tab 在所有上下文中始终可见：

| 编辑器上下文 | 显示的 Tab 列表 |
|-------------|----------------|
| 未选中元素 | 设计、切换、动画、**AI** |
| 选中单个元素 | 样式、位置、动画、**AI** |
| 选中多个元素 | 样式、位置、**AI** |

特殊规则：当用户处于 AI tab 时，选中/取消选中元素不会自动切走，保持在 AI tab。

### 2.3 侧边栏面板（AI tab 内容）

AI tab 被选中时，侧边栏显示以下内容（从上到下）：

1. **快捷输入区**
   - 输入框 + 语音按钮 + 发送按钮
   - 输入框 placeholder："给 AI 发指令..."
   - 下方提示文字：根据是否选中元素动态变化
     - 选中元素时："已选中元素，AI 将直接操作该元素"
     - 未选中时："未选中元素，AI 将生成内容"

2. **新建对话按钮**
   - 虚线边框按钮，点击创建空会话并打开悬浮框

3. **历史记录列表**
   - 标题"历史记录"
   - 每条记录显示：图标（对话/元素调整）+ 标题 + 消息数 + 时间
   - 点击打开该会话的悬浮对话框
   - 当前活跃会话高亮显示
   - 无记录时显示"暂无对话记录"

### 2.4 悬浮对话框 [Updated 2026-04-24]

点击历史记录或新建对话时，弹出悬浮对话框：

| 属性 | 值 |
|------|-----|
| 宽度 | 440px |
| 最大高度 | 720px |
| z-index | 9999（最高层级，不被任何元素遮挡） |
| 初始位置 | 屏幕右侧偏左（`window.innerWidth - 560, 80`） |
| 可拖动 | 是，通过标题栏拖动，限制不超出屏幕 |
| 关闭方式 | 点击关闭按钮，或点击悬浮框外部区域 |
| 关闭后 | 会话保留，可从历史记录重新打开 |

对话框内字号规格：

| 区域 | 字号 |
|------|------|
| 标题栏 | 14px |
| 消息气泡（用户/AI） | 14px |
| 欢迎文字 | 14px |
| 操作按钮 | 12px |
| 输入框 | 14px |
| 工具栏/标签/设置面板 | 12px |
| PPT 内容预览徽章 | 12px |

## 第 3 章：对话交互

### 3.1 消息发送流程 [Updated 2026-04-17]

```
用户输入消息（文字或语音）
  │
  ├── 从侧边栏底部输入 → 创建会话 → 打开悬浮框 → 发送
  └── 从悬浮框输入 → 直接发送
        │
        ↓
  ── 第一层：本地指令解析（0 token，<50ms）──
  正则匹配：字号/颜色/加粗/对齐/尺寸/位置/背景等
        │
        ├── 匹配成功 → 直接执行 → 回复"已调整" + 撤销按钮
        │
        └── 匹配失败 → 进入第二层
              │
              ↓
  ── 第二层：LLM Function Calling ──
  注入幻灯片上下文（选中元素用→标记）
  调用 LLM（携带 7 个工具定义）
        │
        ├── tool_calls → 执行工具 → 流式总结 → 撤销按钮
        └── 纯文本 → 直接显示 → 插入按钮
```

### 3.2 消息气泡

| 类型 | 样式 | 内容渲染 |
|------|------|----------|
| 用户消息 | 蓝色背景，右对齐 | 纯文本 |
| AI 消息 | 灰色背景，左对齐 | Markdown 渲染（标题/列表/加粗/代码块/表格/引用） |
| AI 思考中 | 灰色背景，左对齐 | 三个弹跳圆点动画 |

AI 消息的 Markdown 渲染规则：
- `# 标题` → 1.4em 加粗
- `## 副标题` → 1.2em 加粗
- `**加粗**` → 粗体
- `- 列表` → 带缩进的列表
- `` `代码` `` → 灰底等宽字体
- `> 引用` → 左边框灰色引用块
- 表格 → 带边框的表格

### 3.3 操作按钮 [Updated 2026-04-24]

每条 AI 回复下方显示操作按钮，按钮内容取决于 AI 回复的内容类型标记：

| 内容类型标记 | 含义 | 显示的按钮 |
|-------------|------|-----------|
| `---CONTENT_TYPE:chat---` | 闲聊/问答/解释说明 | 无按钮 |
| `---CONTENT_TYPE:slide---` | 适合插入幻灯片正文的内容 | "插入当前页"、"新建页插入" |
| `---CONTENT_TYPE:note---` | 适合作为演讲稿/备注的内容 | "插入为备注"、"新建页插入" |
| 执行了工具操作 | 修改元素/背景等 | "撤销" |

LLM 在每条回复末尾附加内容类型标记，前端解析后决定按钮。闲聊内容不再显示多余的插入按钮。

"插入为备注"操作：将内容（去除 Markdown 格式）追加到当前页的演讲者备注区域。

### 3.4 多页内容生成 [Added 2026-04-24]

当用户要求生成多个独立内容（如"生成三道习题"），AI 使用 `---PPT_SLIDE---` 分隔符将每页内容独立输出。前端解析后生成多个独立的内容卡片：

- 每个卡片有自己的"插入当前页"/"新建页插入"按钮
- 卡片右上角显示页码标签（如 `1/3`、`2/3`、`3/3`）
- 用户可以逐个选择要不要插入，不必全部接受

### 3.4 语音输入

| 属性 | 说明 |
|------|------|
| 技术 | Web Speech API（`webkitSpeechRecognition`） |
| 语言 | 中文（zh-CN） |
| 入口 | 侧边栏输入框旁 + 悬浮框输入框旁 |
| 按钮状态 | 默认：麦克风图标；录音中：红色方块 + 脉冲动画 |
| 行为 | 点击开始录音，再次点击停止；识别结果实时填入输入框 |
| 降级 | 浏览器不支持时隐藏语音按钮 |

### 3.5 工具选择

悬浮对话框底部有"工具"按钮，hover 弹出工具菜单：

| 工具名称 | 预设 Prompt |
|----------|------------|
| 生成课堂引入 | "请结合情境导入方式，设计一段适中难度的课堂引入" |
| 搜索背景知识 | "搜索简单难度的背景知识" |
| 例题生成 | "生成2题选择题，难度基础" |
| 互动环节设计 | "设计一个小组讨论互动环节，时长10分钟" |
| 总结要点 | "帮我总结全部课件的知识要点，格式：要点列表" |
| 生成演讲稿 | "请生成一份正式风格、5分钟的演讲稿" |

选择工具后：
1. 预设 prompt 自动填入输入框
2. 输入框获得焦点，用户可修改后发送
3. 工具标签显示在工具栏，可点击 × 取消

工具菜单交互：hover 打开，150ms 延迟关闭（防止鼠标移动间隙丢失）。

## 第 4 章：AI 能力

### 4.0 本地指令解析器 [Updated 2026-04-26]

选中元素后，以下指令由本地正则匹配直接执行，不调用 LLM：

| 指令类型 | 示例 | 匹配模式 | 需要选中元素 |
|----------|------|----------|-------------|
| 字号 | "字号改成30"、"字号大一点" | `/字号[改调设]?[成整为到]*\s*(\d+)/` | 是 |
| 文字颜色 | "颜色改红色"、"改成#ff0000" | 30+ 中英文颜色名 + hex | 是 |
| 加粗/斜体 | "加粗"、"取消加粗"、"斜体" | 关键词匹配 | 是 |
| 对齐 | "居中"、"左对齐"、"右对齐" | 关键词匹配 | 是 |
| 尺寸 | "宽度改成500"、"放大"、"缩小" | 数值提取 / ×1.2 / ×0.8 | 是 |
| 位置 | "水平居中"、"垂直居中" | 计算居中坐标 | 是 |
| 背景色 | "背景改蓝色" | 颜色解析 | 否（页面级） |
| 填充/透明度/行高 | "透明度改成50"、"行高1.8" | 数值提取 | 是 |
| 删除 | "删除"、"移除" | 关键词匹配 | 是 |
| 优化布局 | "优化布局"、"调整布局"、"排版优化" | 关键词匹配 → 页面级布局优化 | 否（页面级）⚠️ |

> ⚠️ **已知问题**：布局优化指令虽然是页面级操作（不需要选中元素），但 `tryLocalCommand` 的调用入口有 `selectedElId` 前置条件。当用户未选中任何元素时说"优化布局"，本地指令不会被触发，会走 LLM 路径。需要修复调用入口，将布局优化和背景色等页面级指令从 `selectedElId` 条件中独立出来。

**选区感知格式修改**：当用户双击文本框进入编辑模式并选中部分文字时，字号/颜色/加粗/斜体/对齐等格式指令只对选中的文字生效（通过 ProseMirror emitter 命令实现）。未进入编辑模式时，格式指令对整个文本元素生效（通过 HTML 正则替换实现）。

**布局优化指令**：分析当前页面所有文本元素，判断页面类型后按规则调整：

| 页面类型 | 判断条件 | 调整规则 |
|----------|---------|---------|
| 目录页 | 3+ 个短文本（每个 < 40 字） | 全部 40px 居中，宽度 900px（左右各 50px 边距），垂直均匀分布（每项高度 60px） |
| 标题+内容页 | 首个文本 < 50 字 + 后续有长内容 | 标题 40px（高度 68px），内容默认 32px（超出时每次 -2px 缩到最小 24px），宽度 = 画布宽 - 元素左坐标 - 右边距(50) |
| 其他 | 不符合以上条件 | 只修正超出画布的宽度；若所有元素已在画布内则回复"布局已合理，无需调整" |

### 4.1 上下文注入 [Updated 2026-04-17]

每次用户发送消息时，系统自动在用户消息前注入当前幻灯片的结构化描述。选中元素用 `→` 标记并附带详细属性：

```
[当前幻灯片]
第3/15页 画布:1000×562px
背景:#ffffff
  [文本 id="abc123"] (60,50) 880×52 "Section A: Reading..."
→ [文本 id="def456"] (60,120) 880×200 字号:18px 颜色:#333 对齐:left 行高:1.5 "正文内容..."
  [图片 id="ghi789"] (100,350) 400×180

[用户请求]
把标题字号改大一些
```

### 4.2 Function Calling 工具 [Updated 2026-04-23]

AI 通过 OpenAI 兼容的 Function Calling 机制操作 PPT。系统定义了 7 个工具：

| 工具 | 用途 | 关键参数 | 边界约束 |
|------|------|----------|----------|
| `update_element` | 修改已有元素 | element_id（必填）, left, top, width, height, content, fill, opacity, rotate, lineHeight, paragraphSpace | left/top 自动 clamp 保证元素至少 10% 可见；width/height 限制在 10~1000/562px |
| `add_text_element` | 添加文本 | content（必填）, left, top, width, height, fontSize, color, align, bold, fill, lineHeight | width 自动 clamp 不超出画布右边界 |
| `add_image_element` | 添加图片 | src（必填）, left, top, width, height, radius | — |
| `add_slide` | 新建页面 | background_color, remark, elements[]（每个元素含 type/位置/内容） | — |
| `delete_element` | 删除元素 | element_id（必填） | — |
| `update_slide_background` | 改背景色 | color（必填） | — |
| `update_slide_remark` | 改备注 | remark（必填） | — |

画布坐标系：宽 1000px，高 562px（16:9），左上角为原点。

### 4.3 LLM 调用策略

| 场景 | 请求方式 | 说明 |
|------|----------|------|
| 首次请求 | 非流式 + tools | 检测 LLM 是否返回 tool_calls |
| 有 tool_calls | 执行工具 → 流式请求获取总结 | 两次请求 |
| 无 tool_calls | 直接使用首次请求的文本 | 一次请求，速度快 |

### 4.4 布局优化约束 [Updated 2026-04-26]

当用户要求"优化布局"或"调整布局"时：

**优先由本地指令处理**（零延迟），匹配正则 `/优化.{0,2}布局|调整.{0,2}布局|排版优化|优化.{0,2}排版|整理.{0,2}布局/`，分析页面类型后按规则调整。

如果本地指令未拦截到（用户用了其他表述），LLM 按以下规则操作：

| 步骤 | 说明 |
|------|------|
| 分析页面类型 | 目录页（3+ 短文本）/ 标题+内容页 / 其他 |
| 目录页调整 | 全部 40px 居中，宽度 900px，垂直均匀分布 |
| 标题+内容页调整 | 标题 40px，内容 24-32px 自适应，宽度 = 画布宽 - 左坐标 - 右边距 |
| 边界安全 | left + width ≤ 1000px，不改变文本内容 |

### 4.5 教育理论框架 [Updated 2026-04-26]

AI 生成的所有教学内容遵循以下教育理论，自然融入而非生硬标注：

| 教学法 | 核心原则 | 适用工具 |
|--------|---------|---------|
| POA 产出导向法 | 驱动（真实场景激发需求）→ 促成（提供支架）→ 评价（检验产出） | 课堂引入、例题、互动、总结 |
| PBL 项目式学习 | 真实问题驱动，团队协作，成果展示 | 互动环节设计 |
| 场景教学法 | 创设真实语言使用场景，强调交际功能 | 课堂引入、背景知识、例题、演讲稿 |

各工具与教学法的对应关系：

| 工具 | 主要教学法 | 设计要点 |
|------|-----------|---------|
| 生成课堂引入 | POA 驱动 + 场景教学法 | 创设真实交际场景，让学生意识到"我需要学这个" |
| 搜索背景知识 | 场景教学法 | 选择能创设真实语言使用场景的素材 |
| 例题生成 | 场景教学法 + POA 促成 | 每题嵌入真实交际场景，体现"理解→模仿→应用"梯度 |
| 互动环节设计 | PBL + POA 产出 | 以真实问题/微型项目驱动，明确最终产出形式 |
| 总结要点 | POA 评价 | 按"驱动回顾→知识梳理→产出检验"组织，标注"能做什么" |
| 生成演讲稿 | 场景教学法 + POA 产出 | 演讲稿本身是产出任务，融入具体场景和案例 |

### 4.6 内容插入宽度约束 [Added 2026-04-24]

通过 AI 插入的所有文本元素，宽度自动约束不超出画布右边界：

| 插入方式 | 宽度计算规则 |
|---------|-------------|
| buildTemplatedSlide（对话插入） | width = 画布宽(1000) - 元素左坐标 - 右边距(50) |
| add_text_element（LLM 工具） | width = min(指定宽度, 1000 - left) |
| add_slide 中的文本元素 | 同上 |

### 4.7 自适应字号 [Added 2026-04-24]

通过 AI 对话插入的内容，字号根据内容长度和可用空间自动调整：

| 元素 | 默认字号 | 最小字号 | 自适应规则 |
|------|---------|---------|-----------|
| 标题 | 40px | 28px | 估算文本渲染宽度，逐步缩小直到单行放下 |
| 正文 | 32px | 24px | 同时检查最长行不折行 + 总高度不超出可用区域 |

文本宽度估算：CJK 字符 ≈ 1.0×fontSize，Latin 字符 ≈ 0.55×fontSize，加粗 +10%。

## 第 5 章：内容插入规则

### 5.1 Markdown → PPT 元素转换

用户点击"插入当前页"或"新建页插入"时，AI 回复的 Markdown 文本被解析为 PPT 元素：

**标题识别规则**：
- 首行匹配 `# 标题` 或 `## 标题` → 提取为标题
- 首行短于 50 字符且后续有更多内容 → 视为标题
- 其余情况 → 无标题，全部作为正文

**元素生成规则**（最多 2 个元素：标题 + 正文）：

| 元素 | 位置 | 尺寸 | 字号 | 样式 |
|------|------|------|------|------|
| 标题 | left:60, top:50 | 880×52 | 28px | 加粗，颜色 #1a1a2e |
| 正文 | left:60, top:118 | 880×(剩余高度) | 自适应 | 颜色 #333，行高 1.6 |

**正文字号自适应** [Updated 2026-04-24]：

| 条件 | 字号 |
|------|------|
| 默认（内容不超出） | 32px |
| 内容较多，超出可用区域 | 逐步缩小（每次 -2px） |
| 最小值 | 24px |

自适应同时检查横向（最长行不折行）和纵向（总高度不超出画布底部）。

**正文 Markdown 格式保留**：

| Markdown 语法 | PPT HTML 转换 |
|---------------|--------------|
| `## 副标题` | 字号+2px，加粗，颜色 #1a1a2e |
| `- 列表项` | 前缀 `•`，缩进 1em |
| `1. 有序列表` | 前缀 `•`（简化处理） |
| `**加粗**` | `<b>` 标签 |
| `*斜体*` | `<i>` 标签 |
| 空行 | 跳过（不生成额外空段落） |
| 普通段落 | 标准 `<p>` 标签 |

### 5.2 "新建页插入"

创建一个新的幻灯片（白色背景），包含上述标题+正文元素，插入到当前页之后。

### 5.3 "插入当前页"

将标题和正文元素添加到当前选中的幻灯片上（不删除已有元素）。

### 5.4 "撤销" [Updated 2026-04-23]

直接调用编辑器的 `useHistorySnapshot().undo()` 方法，回退 AI 工具调用造成的修改。

> ⚠️ 旧实现通过 `document.dispatchEvent(new KeyboardEvent(...))` 模拟 Ctrl+Z，但由于悬浮对话框获得焦点时 `editorAreaFocus` 为 false，全局快捷键处理器会忽略该事件，导致撤销无效。已改为直接调用 undo API。

## 第 6 章：会话管理

### 6.1 会话数据结构

每个会话包含：
- 唯一 ID
- 标题（自动从首条用户消息截取前 20 字符）
- 绑定的元素 ID（可选，用户选中元素后从侧边栏发起时绑定）
- 消息列表
- 创建时间

### 6.2 会话生命周期

```
创建 ──→ 活跃（悬浮框打开）──→ 挂起（悬浮框关闭）──→ 恢复（重新打开）
```

- 创建：从侧边栏输入发送 / 点击"新建对话"
- 挂起：关闭悬浮框，会话数据保留
- 恢复：从历史记录点击，重新打开悬浮框
- 持久化：每次消息变更后保存到 sessionStorage

### 6.3 元素绑定

当用户选中一个元素后从侧边栏快捷输入发消息：
- 会话自动绑定该元素 ID
- 历史记录中显示"编辑"图标（区别于普通对话图标）
- 悬浮框中显示"绑定元素"标识
- AI 上下文中包含该元素的详细信息

## 第 7 章：极限场景与异常处理

### A. 内容溢出

| 场景 | 当前处理 | 产品要求 |

| 历史记录过多 | sessionStorage 无上限 | 需补充：限制最大会话数或提供清理 |

### B. 空内容/无数据

| 场景 | 当前处理 | 产品要求 |
|------|----------|----------|
| 无历史记录 | 显示"暂无对话记录" | 已处理 |
| AI 返回空内容 | 显示 thinking 动画后空气泡 | 需补充：显示"未获取到有效回复"提示 |
| 新会话无消息 | 悬浮框显示"有什么需要帮忙的？" | 已处理 |

### C. 网络异常

| 场景 | 当前处理 | 产品要求 |
|------|----------|----------|
| LLM API 不可用 | 显示"服务暂时不可用" | 已处理 |
| 网络断开 | 显示"网络错误，请重试" | 已处理 |
| 流式响应中断 | 显示已接收的部分内容 | 需补充：添加重试按钮 |

### D. 用户操作异常

| 场景 | 当前处理 | 产品要求 |
|------|----------|----------|
| 流式输出中重复发送 | 发送按钮 disabled | 已处理 |
| 工具菜单 hover 间隙 | 150ms 延迟 + CSS 桥接 | 已处理 |
| 悬浮框拖出屏幕 | Math.max/min 限制 | 已处理 |
| 浏览器刷新 | sessionStorage 保留会话 | 已处理（关闭浏览器丢失） |

## 第 10 章：数据埋点

### 10.1 核心指标

| 指标 | 定义 | 计算方式 |
|------|------|----------|
| AI 使用率 | 使用 AI 的用户占比 | 发送过消息的用户 / 总活跃用户 |
| 内容采纳率 | 点击"插入"的比例 | 插入次数 / AI 回复总数 |
| 工具调用成功率 | Function Calling 成功率 | 成功数 / 调用总数 |

### 10.2 用户行为埋点

| 行为 | 事件名 | 参数 |
|------|--------|------|
| 发送消息 | `ai_send_message` | 消息长度、是否绑定元素、使用的工具 |

| 撤销操作 | `ai_undo` | 撤销的工具名称 |
| 语音输入 | `ai_voice_input` | 识别文本长度 |
| 打开会话 | `ai_open_session` | 新建/历史 |
| 选择工具 | `ai_select_tool` | 工具名称 |

### 10.3 业务事件埋点

| 事件 | 事件名 | 参数 |
|------|--------|------|
| 工具调用 | `ai_tool_call` | 工具名、执行结果 |
| 响应完成 | `ai_response_done` | 响应时长、是否含工具调用 |
