# AI PPT 编辑器 — 项目背景与规划上下文

本文件记录项目的真实业务背景、组织依赖关系和规划策略，避免每次对话重复说明。

## 项目定位

这是一个 AI 备课工具，帮助大学英语教师高效制作课件。当前以 PPTist 开源项目为基座，叠加 AI 助手"子言"能力。

## 当前状态

### Demo vs 生产的区别
- 当前 demo 使用 mock 数据，纯前端运行
- 生产环境需要服务端配合（API 代理、会话持久化、素材库接口等）

### 素材库依赖
- 社里有素材库资源，但需要社里提供接口/数据
- AI 备课工具自己没有素材库，依赖社里配合
- 图片搜索当前用 Pexels 免费 API，生产环境需替换为社里的素材库或付费 API

## 规划策略优先级

### 第一优先：体验提速
- AI 对话响应速度优化
- 本地指令零延迟执行
- 内容插入的模板继承和自适应
- 这是最重要的，直接影响老师的使用意愿

### 第二优先：智能深化（我们自己能做的）
- 内容审核优化（生成内容的质量把控）
- 内容润色优化（语言表达、教学法融入）
- 教育理论框架深化（POA/PBL/场景教学法）
- 文生图能力（作为搜索图片的补充）
- 这些不依赖外部，团队自己可以推进

### 第三优先：生态打通（依赖社里配合）
- 接入社里的素材库（图片、音视频、教学资源）
- 编辑审核流程对接
- 这些需要社里提供支持，推进节奏不完全由我们控制

## 目标用户画像
- 大学英语教师和教研人员
- 不熟悉 PPT 排版细节
- 希望通过自然语言完成内容生成和格式调整
- 教学内容需要符合教育理论（POA、PBL、场景教学法）
- 可能需要离线使用（学校电脑可能没有联网）

## 技术决策记录
- 基座：PPTist 开源项目（Vue 3 + TypeScript）
- LLM：qwen-plus（通过 modelproxy.unipus.cn 代理）
- 图片搜索：Pexels API（临时方案，待替换）
- 会话存储：sessionStorage（临时方案，生产需后端持久化）
- 离线方案：导出 PPTX 是最现实的方案，离线放映包需额外开发
