## 文档目的

本产品基于开源项目 PPTist 二次开发，在 Web 端演示文稿编辑器的基础上，集成了 AI 助手能力。AI 助手（"子言"）可以通过自然语言对话帮助用户生成课件内容、调整元素格式、插入新页面，并通过 LLM Function Calling 直接操作 PPT 元素。本文档是 demo 代码的反向工程产物，用于研发 AI 工具的需求输入。

## 第 1 章：产品概述

### 1.1 产品描述

PPT AI Agent 是一个基于 Web 的在线演示文稿（PPT）编辑与预览工具，内置 AI 助手"子言"。产品解决教师在制作课件时的效率问题：通过自然语言指令，AI 可以自动生成课堂引入、例题、演讲稿等教学内容，并直接插入或调整 PPT 中的元素格式（字号、颜色、位置、对齐等），无需手动逐项操作。

核心能力：
- 完整的 PPT 在线编辑（文字、图片、形状、线条、图表、表格、视频、音频、公式）
- AI 助手通过 LLM Function Calling 直接操作 PPT 元素
- 多会话管理，支持元素绑定的上下文对话
- 内容智能插入（Markdown 解析 → PPT 元素自动布局）
- 幻灯片放映与导出（PPTX/JSON/PDF/图片）

### 1.2 目标用户

- 中小学及高校教师（主要用户），需要高效制作英语/学科课件
- 教研人员，需要批量生成教学素材
- 培训讲师，需要快速制作演示文稿

## 第 3 章：功能逻辑

### 3.1: AI 助手（子言）

#### 3.1.1 功能目的

让教师通过自然语言与 AI 对话，完成课件内容生成和元素格式调整，降低 PPT 制作门槛。

#### 3.1.2 操作流程

```
用户点击右侧边栏"AI"tab
  │
  ↓
显示 AI 侧边栏（历史记录 + 快捷输入框）
  │
  ├── 用户在快捷输入框输入 → 创建新会话 → 弹出悬浮对话框 → 自动发送消息
  ├── 用户点击"新建对话" → 弹出空的悬浮对话框
  └── 用户点击历史记录 → 弹出该会话的悬浮对话框
        │
        ↓
  悬浮对话框中用户输入消息
        │
        ↓
  系统注入当前幻灯片上下文（元素ID、类型、位置、内容摘要）
        │
        ↓
  调用 LLM API（带 Function Calling tools）
        │
        ├── LLM 返回 tool_calls → 执行工具操作（修改元素/添加元素/新建页等）
        │     │
        │     ↓
        │   再次调用 LLM 获取操作总结 → 流式显示 → 显示"撤销"按钮
        │
        └── LLM 返回纯文本 → 直接显示 → 显示"插入当前页/新建页插入"按钮
```

#### 3.1.3 状态机

```
┌──────────┐  输入消息  ┌──────────┐  LLM响应  ┌──────────┐
│  空闲     │ ────────→ │ 流式输出  │ ────────→ │ 完成     │
│  idle    │           │ streaming│           │ done     │
└──────────┘           └──────────┘           └──────────┘
                                                   │
                                              显示操作按钮
```

| 状态 | UI 表现 | 可执行操作 |
|------|---------|-----------|
| idle | 输入框可用，发送按钮可点击 | 输入文字、语音输入、选择工具 |
| streaming | 输入框禁用，显示 thinking 动画或流式文字 | 无（等待完成） |
| done | 显示完整回复 + 操作按钮 | 插入内容、撤销、继续对话 |

#### 3.1.4 交互规则

| 元素 | 交互方式 | 触发行为 | 前置条件 |
|------|----------|----------|----------|
| 快捷输入框 | 回车 | 创建会话 + 打开悬浮框 + 自动发送 | 输入非空 |
| 语音按钮 | 点击 | 开始/停止语音识别（Web Speech API） | 浏览器支持 |
| 工具选择 | hover 打开菜单 → 点击选项 | 填入预设 prompt 到输入框 | 无 |
| 历史记录项 | 点击 | 打开该会话的悬浮对话框 | 无 |
| 悬浮框标题栏 | 拖动 | 移动悬浮框位置 | 无 |
| 悬浮框关闭按钮 | 点击 | 关闭悬浮框（会话保留） | 无 |
| "插入当前页"按钮 | 点击 | 将 AI 回复内容解析为 PPT 元素插入当前页 | AI 回复完成 |
| "新建页插入"按钮 | 点击 | 新建幻灯片并插入内容 | AI 回复完成 |
| "撤销"按钮 | 点击 | 触发 Ctrl+Z 撤销上一步操作 | 工具调用完成后 |

#### 3.1.5 AI 工具能力（Function Calling）

| 工具名称 | 功能 | 参数 |
|----------|------|------|
| update_element | 修改已有元素属性 | element_id, left, top, width, height, content, fill, opacity, rotate, lineHeight |
| add_text_element | 添加文本元素 | left, top, width, height, content, fontSize, color, align, bold, fill |
| add_image_element | 添加图片元素 | src, left, top, width, height, radius |
| add_slide | 新建幻灯片 | background_color, remark, elements[] |
| delete_element | 删除元素 | element_id |
| update_slide_background | 修改背景色 | color |
| update_slide_remark | 修改备注 | remark |

#### 3.1.6 极限场景与异常处理

**A. 内容溢出场景**

| 场景 | 当前 demo 处理 | 产品要求 |
|------|---------------|----------|
| AI 回复内容过长 | 悬浮框内滚动显示，max-height 480px | 保持滚动，自动滚到底部 |
| 插入内容超出幻灯片 | 自适应字号（16→12px），超长自动分页 | 需补充：分页时保持标题连贯性 |
| 聊天历史过多 | sessionStorage 存储，无上限 | 需补充：考虑存储上限和清理策略 |

**B. 空内容/无数据场景**

| 场景 | 当前 demo 处理 | 产品要求 |
|------|---------------|----------|
| 无历史记录 | 显示"暂无对话记录"空状态 | 已处理 |
| AI 返回空内容 | 显示 thinking 动画后无文字 | 需补充：显示"AI 未返回有效内容"提示 |

**C. 网络与接口异常场景**

| 场景 | 当前 demo 处理 | 产品要求 |
|------|---------------|----------|
| LLM API 不可用 | 显示"服务暂时不可用，请稍后再试" | 已处理 |
| 网络断开 | 显示"网络错误，请重试" | 已处理 |
| 流式响应中断 | 显示已接收的部分内容 | 需补充：添加重试按钮 |

**D. 用户操作异常场景**

| 场景 | 当前 demo 处理 | 产品要求 |
|------|---------------|----------|
| 流式输出中重复发送 | 发送按钮 disabled | 已处理 |
| 工具菜单 hover 间隙 | 150ms 延迟定时器 + CSS 桥接 | 已处理 |
| 悬浮框拖出屏幕 | Math.max/min 限制范围 | 已处理 |

## 第 4 章：业务规则

### 4.1 内容插入规则

| 规则名称 | 条件描述 | 效果 |
|----------|----------|------|
| Markdown 标题识别 | AI 回复首行匹配 `# 标题` 或短于 50 字符 | 拆为标题元素（28px 加粗）+ 正文元素 |
| 正文字号自适应 | 正文长度 <200 字 → 16px，<400 → 15px，<700 → 14px，≥700 → 12px | 自动调整字号 |
| Markdown 内联格式 | `**加粗**` → `<b>`，`*斜体*` → `<i>`，列表 → `•` 前缀 | 保留格式插入 PPT |
| 空行过滤 | Markdown 空行 | 跳过，不生成额外空段落 |

| 规则名称 | 条件描述 | 效果 |
|----------|----------|------|
| 幻灯片上下文 | 每次用户发消息 | 自动注入当前页元素信息（ID、类型、位置、内容摘要） |
| 元素绑定 | 用户选中元素后从侧边栏发消息 | 会话绑定该元素 ID，AI 优先操作该元素 |
| 工具 prompt | 用户选择工具（如"生成课堂引入"） | 输入框自动填入预设 prompt |

### 4.3 会话管理规则

| 规则名称 | 条件描述 | 效果 |
|----------|----------|------|
| 自动命名 | 会话首条用户消息 | 截取前 20 字符作为会话标题 |
| 会话持久化 | 每次消息发送/接收后 | 保存到 sessionStorage |
| 会话恢复 | 页面刷新或重新打开 AI tab | 从 sessionStorage 恢复所有会话 |

## 第 7 章：数据埋点

### 7.1 核心指标

| 指标名称 | 定义 | 计算方式 |
|----------|------|----------|
| AI 使用率 | 使用 AI 助手的用户占比 | 发送过 AI 消息的用户 / 总活跃用户 |
| AI 内容采纳率 | 用户点击"插入"的比例 | 插入操作次数 / AI 回复总数 |
| 课件完成率 | 用户完成编辑并导出的比例 | 导出次数 / 打开编辑器次数 |
| AI 操作成功率 | Function Calling 成功执行的比例 | 成功执行数 / 工具调用总数 |

### 7.2 页面浏览埋点

| 页面 | 埋点事件名 | 触发时机 |
|------|-----------|----------|
| 编辑器 | `page_editor_view` | 编辑器加载完成 |
| 放映模式 | `page_screen_view` | 进入放映 |
| AI 侧边栏 | `page_ai_sidebar_view` | 切换到 AI tab |

### 7.3 用户行为埋点

| 行为 | 埋点事件名 | 采集参数 |
|------|-----------|----------|
| 发送 AI 消息 | `action_ai_send` | 消息长度、是否绑定元素、使用的工具 |
| 插入 AI 内容 | `action_ai_insert` | 插入方式（当前页/新建页）、内容长度 |
| AI 撤销操作 | `action_ai_undo` | 撤销的操作类型 |
| 语音输入 | `action_voice_input` | 识别文本长度 |
| 导出文件 | `action_export` | 导出格式 |
| 导入文件 | `action_import` | 导入格式、文件大小 |

### 7.4 业务事件埋点

| 事件 | 埋点事件名 | 采集参数 |
|------|-----------|----------|
| AI 工具调用 | `event_ai_tool_call` | 工具名称、参数摘要、执行结果 |
| AI 响应完成 | `event_ai_response_done` | 响应时长、token 数、是否含工具调用 |
| 课件导出成功 | `event_export_success` | 格式、页数、文件大小 |
