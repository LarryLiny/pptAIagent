> 本文档描述 PPT 编辑器中"AI 助手（子言）"功能模块的产品需求。
> 编辑器本身（元素编辑、放映、导入导出等）为系统已有能力，不在本文档范围内。
> AI 助手通过调用编辑器已有的 Store API 来操作 PPT 元素。

---

## 文档目的

AI 助手"子言"是嵌入在线 PPT 编辑器右侧工具栏中的智能对话模块。教师通过自然语言与 AI 对话，AI 可以直接操作 PPT 元素（修改格式、添加内容、新建页面），也可以生成教学内容供用户选择性插入。本文档聚焦 AI 助手的交互逻辑、对话管理、内容生成与插入规则。

---

## 第 1 章：产品概述

### 1.1 功能定位

AI 助手是 PPT 编辑器的增值能力层，不改变编辑器的核心交互。它以"侧边栏 tab + 悬浮对话框"的形态存在，用户可以随时在编辑属性和 AI 对话之间切换，互不干扰。

### 1.2 核心场景

| 场景 | 用户行为 | AI 响应 |
|------|----------|---------|
| 生成教学内容 | "帮我设计一段课堂引入" | 生成 Markdown 格式内容，用户可插入 PPT |
| 调整元素格式 | 选中文字框 → "字号改大，颜色改红" | 直接修改元素属性，回复"已调整"+撤销按钮 |
| 批量操作 | "把背景改成深蓝色" | 调用 update_slide_background 工具 |
| 生成例题 | 选择"例题生成"工具 → 输入需求 | 生成题目内容，可插入当前页或新建页 |
| 生成演讲稿 | 选择"生成演讲稿"工具 | 生成演讲稿，可插入到页面备注 |

### 1.3 目标用户

教师和教研人员，需要高效制作课件，不熟悉 PPT 排版细节，希望通过自然语言完成内容生成和格式调整。

## 第 2 章：入口与布局

### 2.1 AI 助手入口

AI 助手作为右侧工具栏的一个 tab，与"设计/样式/位置/动画"等 tab 并列。

| 位置 | 说明 |
|------|------|
| 右侧工具栏竖向 tab 栏 | 紧挨"动画"tab 下方，有分隔线 |
| 图标 | 星形（sparkle）线框图标，科技感 |
| 标签文字 | "AI"，渐变色（紫→蓝） |

### 2.2 Tab 切换规则

AI tab 在所有上下文中始终可见：

| 编辑器上下文 | 显示的 Tab 列表 |
|-------------|----------------|
| 未选中元素 | 设计、切换、动画、**AI** |
| 选中单个元素 | 样式、位置、动画、**AI** |
| 选中多个元素 | 样式、位置、**AI** |

特殊规则：当用户处于 AI tab 时，选中/取消选中元素不会自动切走，保持在 AI tab。

### 2.3 侧边栏面板（AI tab 内容）

AI tab 被选中时，侧边栏显示以下内容（从上到下）：

1. **快捷输入区**
   - 输入框 + 语音按钮 + 发送按钮
   - 输入框 placeholder："给 AI 发指令..."
   - 下方提示文字：根据是否选中元素动态变化
     - 选中元素时："已选中元素，AI 将直接操作该元素"
     - 未选中时："未选中元素，AI 将生成内容"

2. **新建对话按钮**
   - 虚线边框按钮，点击创建空会话并打开悬浮框

3. **历史记录列表**
   - 标题"历史记录"
   - 每条记录显示：图标（对话/元素调整）+ 标题 + 消息数 + 时间
   - 点击打开该会话的悬浮对话框
   - 当前活跃会话高亮显示
   - 无记录时显示"暂无对话记录"

### 2.4 悬浮对话框

点击历史记录或新建对话时，弹出悬浮对话框：

| 属性 | 值 |
|------|-----|
| 宽度 | 400px |
| 最大高度 | 680px |
| z-index | 9999（最高层级，不被任何元素遮挡） |
| 初始位置 | 屏幕右侧偏左（`window.innerWidth - 520, 80`） |
| 可拖动 | 是，通过标题栏拖动，限制不超出屏幕 |
| 关闭方式 | 点击关闭按钮，或点击悬浮框外部区域 |
| 关闭后 | 会话保留，可从历史记录重新打开 |

## 第 3 章：对话交互

### 3.1 消息发送流程

```
用户输入消息（文字或语音）
  │
  ├── 从侧边栏快捷输入 → 自动创建会话 → 打开悬浮框 → 发送
  └── 从悬浮框输入 → 直接发送
        │
        ↓
系统自动注入当前幻灯片上下文
  │
  ↓
调用 LLM API（携带 Function Calling 工具定义）
  │
  ├── LLM 返回 tool_calls
  │     → 执行工具操作（直接修改 PPT）
  │     → 再次调用 LLM 获取操作总结（流式）
  │     → 显示总结 + "撤销"按钮
  │
  └── LLM 返回纯文本
        → 直接显示（非流式，一次性返回）
        → 显示"插入当前页" + "新建页插入"按钮
```

### 3.2 消息气泡

| 类型 | 样式 | 内容渲染 |
|------|------|----------|
| 用户消息 | 蓝色背景，右对齐 | 纯文本 |
| AI 消息 | 灰色背景，左对齐 | Markdown 渲染（标题/列表/加粗/代码块/表格/引用） |
| AI 思考中 | 灰色背景，左对齐 | 三个弹跳圆点动画 |

AI 消息的 Markdown 渲染规则：
- `# 标题` → 1.4em 加粗
- `## 副标题` → 1.2em 加粗
- `**加粗**` → 粗体
- `- 列表` → 带缩进的列表
- `` `代码` `` → 灰底等宽字体
- `> 引用` → 左边框灰色引用块
- 表格 → 带边框的表格

### 3.3 操作按钮

每条 AI 回复下方显示操作按钮，按钮内容取决于 AI 响应类型：

| AI 响应类型 | 显示的按钮 |
|------------|-----------|
| 执行了工具操作（修改元素/背景等） | "撤销" |
| 生成了文本内容 | "插入当前页"、"新建页插入" |

关键规则：**每条消息的按钮操作的是该条消息自己的内容**，不是全局最后一条。

### 3.4 语音输入

| 属性 | 说明 |
|------|------|
| 技术 | Web Speech API（`webkitSpeechRecognition`） |
| 语言 | 中文（zh-CN） |
| 入口 | 侧边栏输入框旁 + 悬浮框输入框旁 |
| 按钮状态 | 默认：麦克风图标；录音中：红色方块 + 脉冲动画 |
| 行为 | 点击开始录音，再次点击停止；识别结果实时填入输入框 |
| 降级 | 浏览器不支持时隐藏语音按钮 |

### 3.5 工具选择

悬浮对话框底部有"工具"按钮，hover 弹出工具菜单：

| 工具名称 | 预设 Prompt |
|----------|------------|
| 生成课堂引入 | "请结合情境导入方式，设计一段适中难度的课堂引入" |
| 搜索背景知识 | "搜索简单难度的背景知识" |
| 例题生成 | "生成2题选择题，难度基础" |
| 互动环节设计 | "设计一个小组讨论互动环节，时长10分钟" |
| 总结要点 | "帮我总结全部课件的知识要点，格式：要点列表" |
| 生成演讲稿 | "请生成一份正式风格、5分钟的演讲稿" |

选择工具后：
1. 预设 prompt 自动填入输入框
2. 输入框获得焦点，用户可修改后发送
3. 工具标签显示在工具栏，可点击 × 取消

工具菜单交互：hover 打开，150ms 延迟关闭（防止鼠标移动间隙丢失）。

## 第 4 章：AI 能力

### 4.1 上下文注入

每次用户发送消息时，系统自动在用户消息前注入当前幻灯片的结构化描述：

```
[当前幻灯片]
当前是第3/15页，画布尺寸: 1000×562px
背景: 纯色 #ffffff
[文本 id="abc123"] 位置(60,50) 尺寸(880×52) 内容="Section A: Reading and Thinking"
[图片 id="def456"] 位置(100,120) 尺寸(400×300) src="data:image/png;base64..."
[形状 id="ghi789"] 位置(600,200) 尺寸(300×200) (含文字)

[用户请求]
把标题字号改大一些
```

### 4.2 Function Calling 工具

AI 通过 OpenAI 兼容的 Function Calling 机制操作 PPT。系统定义了 7 个工具：

| 工具 | 用途 | 关键参数 |
|------|------|----------|
| `update_element` | 修改已有元素 | element_id（必填）, left, top, width, height, content, fill, opacity, rotate, lineHeight, paragraphSpace |
| `add_text_element` | 添加文本 | content（必填）, left, top, width, height, fontSize, color, align, bold, fill, lineHeight |
| `add_image_element` | 添加图片 | src（必填）, left, top, width, height, radius |
| `add_slide` | 新建页面 | background_color, remark, elements[]（每个元素含 type/位置/内容） |
| `delete_element` | 删除元素 | element_id（必填） |
| `update_slide_background` | 改背景色 | color（必填） |
| `update_slide_remark` | 改备注 | remark（必填） |

画布坐标系：宽 1000px，高 562px（16:9），左上角为原点。

### 4.3 LLM 调用策略

| 场景 | 请求方式 | 说明 |
|------|----------|------|
| 首次请求 | 非流式 + tools | 检测 LLM 是否返回 tool_calls |
| 有 tool_calls | 执行工具 → 流式请求获取总结 | 两次请求 |
| 无 tool_calls | 直接使用首次请求的文本 | 一次请求，速度快 |

System Prompt 要点：
- 角色：专业 PPT 课件制作助手"子言"
- 能力：通过工具直接操作 PPT
- 画布尺寸：1000×562px
- 行为：格式调整用工具直接操作；内容生成用文本回复
- 风格：回复简洁

## 第 5 章：内容插入规则

### 5.1 Markdown → PPT 元素转换

用户点击"插入当前页"或"新建页插入"时，AI 回复的 Markdown 文本被解析为 PPT 元素：

**标题识别规则**：
- 首行匹配 `# 标题` 或 `## 标题` → 提取为标题
- 首行短于 50 字符且后续有更多内容 → 视为标题
- 其余情况 → 无标题，全部作为正文

**元素生成规则**（最多 2 个元素：标题 + 正文）：

| 元素 | 位置 | 尺寸 | 字号 | 样式 |
|------|------|------|------|------|
| 标题 | left:60, top:50 | 880×52 | 28px | 加粗，颜色 #1a1a2e |
| 正文 | left:60, top:118 | 880×(剩余高度) | 自适应 | 颜色 #333，行高 1.6 |

**正文字号自适应**：

| 正文长度 | 字号 |
|----------|------|
| < 200 字符 | 16px |
| 200-400 | 15px |
| 400-700 | 14px |
| ≥ 700 | 12px |

**正文 Markdown 格式保留**：

| Markdown 语法 | PPT HTML 转换 |
|---------------|--------------|
| `## 副标题` | 字号+2px，加粗，颜色 #1a1a2e |
| `- 列表项` | 前缀 `•`，缩进 1em |
| `1. 有序列表` | 前缀 `•`（简化处理） |
| `**加粗**` | `<b>` 标签 |
| `*斜体*` | `<i>` 标签 |
| 空行 | 跳过（不生成额外空段落） |
| 普通段落 | 标准 `<p>` 标签 |

### 5.2 "新建页插入"

创建一个新的幻灯片（白色背景），包含上述标题+正文元素，插入到当前页之后。

### 5.3 "插入当前页"

将标题和正文元素添加到当前选中的幻灯片上（不删除已有元素）。

### 5.4 "撤销"

触发编辑器的撤销操作（模拟 Ctrl+Z 键盘事件），回退 AI 工具调用造成的修改。

## 第 6 章：会话管理

### 6.1 会话数据结构

每个会话包含：
- 唯一 ID
- 标题（自动从首条用户消息截取前 20 字符）
- 绑定的元素 ID（可选，用户选中元素后从侧边栏发起时绑定）
- 消息列表
- 创建时间

### 6.2 会话生命周期

```
创建 ──→ 活跃（悬浮框打开）──→ 挂起（悬浮框关闭）──→ 恢复（重新打开）
```

- 创建：从侧边栏输入发送 / 点击"新建对话"
- 挂起：关闭悬浮框，会话数据保留
- 恢复：从历史记录点击，重新打开悬浮框
- 持久化：每次消息变更后保存到 sessionStorage

### 6.3 元素绑定

当用户选中一个元素后从侧边栏快捷输入发消息：
- 会话自动绑定该元素 ID
- 历史记录中显示"编辑"图标（区别于普通对话图标）
- 悬浮框中显示"绑定元素"标识
- AI 上下文中包含该元素的详细信息

## 第 7 章：极限场景与异常处理

### A. 内容溢出

| 场景 | 当前处理 | 产品要求 |
|------|----------|----------|
| AI 回复过长 | 悬浮框消息区滚动（max-height 480px） | 已处理 |
| 插入内容超出幻灯片 | 字号自适应缩小（16→12px） | 需补充：超长内容自动分页 |
| 历史记录过多 | sessionStorage 无上限 | 需补充：限制最大会话数或提供清理 |

### B. 空内容/无数据

| 场景 | 当前处理 | 产品要求 |
|------|----------|----------|
| 无历史记录 | 显示"暂无对话记录" | 已处理 |
| AI 返回空内容 | 显示 thinking 动画后空气泡 | 需补充：显示"未获取到有效回复"提示 |
| 新会话无消息 | 悬浮框显示"有什么需要帮忙的？" | 已处理 |

### C. 网络异常

| 场景 | 当前处理 | 产品要求 |
|------|----------|----------|
| LLM API 不可用 | 显示"服务暂时不可用" | 已处理 |
| 网络断开 | 显示"网络错误，请重试" | 已处理 |
| 流式响应中断 | 显示已接收的部分内容 | 需补充：添加重试按钮 |

### D. 用户操作异常

| 场景 | 当前处理 | 产品要求 |
|------|----------|----------|
| 流式输出中重复发送 | 发送按钮 disabled | 已处理 |
| 工具菜单 hover 间隙 | 150ms 延迟 + CSS 桥接 | 已处理 |
| 悬浮框拖出屏幕 | Math.max/min 限制 | 已处理 |
| 浏览器刷新 | sessionStorage 保留会话 | 已处理（关闭浏览器丢失） |

### 9.1 简化逻辑

| 功能 | Demo 实现 | 生产要求 |

## 第 10 章：数据埋点

### 10.1 核心指标

| 指标 | 定义 | 计算方式 |
|------|------|----------|
| AI 使用率 | 使用 AI 的用户占比 | 发送过消息的用户 / 总活跃用户 |
| 内容采纳率 | 点击"插入"的比例 | 插入次数 / AI 回复总数 |
| 工具调用成功率 | Function Calling 成功率 | 成功数 / 调用总数 |

### 10.2 用户行为埋点

| 行为 | 事件名 | 参数 |
|------|--------|------|
| 发送消息 | `ai_send_message` | 消息长度、是否绑定元素、使用的工具 |
| 插入内容 | `ai_insert_content` | 插入方式、内容长度 |
| 撤销操作 | `ai_undo` | 撤销的工具名称 |
| 语音输入 | `ai_voice_input` | 识别文本长度 |
| 打开会话 | `ai_open_session` | 新建/历史 |
| 选择工具 | `ai_select_tool` | 工具名称 |

### 10.3 业务事件埋点

| 事件 | 事件名 | 参数 |
|------|--------|------|
| 工具调用 | `ai_tool_call` | 工具名、执行结果 |
| 响应完成 | `ai_response_done` | 响应时长、是否含工具调用 |
